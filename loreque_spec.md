## プロジェクト概要

**ロレクエ** = ロレックス抽選代行コミュニティ **2期生専用** の管理ツール。
1期生用「ロレポチ」（`/Users/ishibashinoriko/Desktop/ロレポチ/`、ahaha48.github.io/lorepochi/）のフォーク。
1期生ツールはそのまま並行稼働中。**ロレクエから1期生のデータ・ファイルには一切触れない。**

## ストック報酬の仕様（2期生の新機能）

```
累積報酬 = 報酬発生週の数 × ¥1,000
  ・報酬発生週 = result_ss=true かつ won_date なし の週
  ・対象: 未当選の方（初当選〔同行補助モードを除く〕より前の週）
          または no_purchase=true の方（全期間）
  ・累積は過去実績から算出 → 当選してステータスが変わっても発生済み分は消えない

支払済 = lq_reward_payments のその方の amount 合計
ストック残 = 累積報酬 − 支払済
```

- **当選したら**: 当選登録モーダルにストック残高が表示される → 当選登録後、「報酬ストック管理」ページの「💴 支払い登録」でまとめて支払いを記録
- 支払い登録: 金額（デフォルト=ストック残全額・編集可）、支払日、備考
- 支払い取消: 支払い履歴の「取消」ボタン → レコード削除でストック残に戻る
- 表示場所: ダッシュボード（未払いストック合計）／週次入力行（💰ストック¥X）／メンバー一覧（ストック残・支払済列）／報酬ストック管理ページ

## セットアップ手順

1. **Supabase**: SQL Editor で `loreque_schema.sql` を全文実行（lq_ テーブル7つが作成される。既存テーブルには無影響）
2. **動作確認**: `index.html` をブラウザで開く → Googleログイン → メンバー0名の状態で起動
3. **公開（任意）**: GitHub に新リポジトリ（例: loreque）を作成して GitHub Pages で公開
   - push 前にセキュリティレビュー必須（グローバル CLAUDE.md ルール）

## セキュリティ

- `index.html` に埋め込まれている SUPABASE_ANON_KEY は **RLS 有効が大前提で公開許容**のキー。
  全 lq_ テーブルで RLS が有効＋authenticated のみ操作可になっていることを必ず確認すること。
  RLS が無効のテーブルが1つでもあると anon key で誰でも読み書きできてしまう。
- メンバーデータ入りのバックアップ JSON（`loreque_backup_*.json`）は個人情報を含むため
  `.gitignore` で除外済み。コミットしないこと。

## 既知の制約（1期生から踏襲）

- **month は年情報を持たない**（1〜12の整数）。年をまたいで運用すると、ストック累積計算の
  「初当選より前の週」判定（month×10+week の大小比較）が 12月→1月 で逆転する。
  年をまたぐ際はデータの持ち方の見直しを検討すること。
- メンバー0名の間は Supabase からのロードが「データなし」扱いになり、毎回「新規データで開始します」と表示される（最初のメンバー追加後は正常）。

## ロールバック手順

- **ローカル**: 「ロレクエ」フォルダをゴミ箱へ移動（`rm -rf` は使わない。Finder か `mv ~/.Trash/`）
- **Supabase**: `loreque_schema.sql` 末尾のコメントアウトされた DROP TABLE 7行を実行（lq_ のみ。1期生テーブルには触れない）
- ロレポチ側（フォルダ・テーブル）には何も書き込んでいないため、1期生への影響はゼロ
